Theme Details

Created: 11 March 2014
Latest Update: 12 March 2014
Update Link: https://wrapbootstrap.com/support/download-resender
By: Around25

ArkAdmin is an clean, flat and beautiful dashboard admin theme that uses the latest version of Bootstrap 3 and jQuery in order to illustrate a simple yet elegant design for managing any kind of web application.

It's focus is on simplifying the UI and delivering the best experience for its users.

It introduces an elegant flat and responsive designed crafted to deliver the best possible performance on any type of device. It's outstanding simple architecture allows any developer to jump right into the code without the need of searching through thousands of lines of javascript/html code.

We worked hard to incorporate any design elements directly on top of the Bootstrap 3 code, in order to allow our users to use the same structure and classes they already know.

In this documentation we assume that you have already read and are familiar with the Bootstrap 3 components and widget structure. If that is not the case for you then please read the Bootstrap 3 Documentation first.

Folder Structure

How is the theme organized?

The theme uses NodeJs to build all the pages, css files, and minimize the scripts.
You don't need to use nodejs in order to use the theme. We used it because it makes it easy to update the files and compile the final theme.

In the public/ folder you will find all the files that you need to create your application design.
In the src/ folder you will find the sources for the CSS files in LESS format, the source code for the JS script (ark.js) and the source for the html pages.

Layout Options

Integrated layout preferences

ArkAdmin contains several options for managing how your web application will look like.

You can have full responsive behaviour, have a floating header and contain your application within a container.

In order use the full responsive behaviour you can just use the code that is contained within the archive.

If you want to have a floating header you can add the fixed_header class to the body tag.

<body class="cover fixed_header">
    <div class="wrapper">
        ...
    </div>
</body>

In case you decide to go with the top navigation in your project and you can add the fixed_menu class to the body tag to make the navigation float over the page.

<body class="cover fixed_menu">
    <div class="wrapper">
        ...
    </div>
</body>

If you want to contain the application within a container just add the container class to the wrapper element.

<body class="cover">
    <div class="wrapper container">
        ...
    </div>
</body>

Icons and Images

How can I display icons or wrap images?

With ArkAdmin adding icons in your application is as easy as adding a simple element to your page.
The theme uses FontAwesome to allow you to use icons easily and combine them in any way you need to.
The easiest way to display an icon is to add a few classes to the <i></i> tag.

<i class="fa fa-bell"></i>

For more details on how you can combine these icons and add additional effects please visit FontAwesome Examples.

Apart from the FontAwesome icons you can also use Glyphicons in a similar fashion.

<span class="glyphicon glyphicon-asterisk"></span>

In order to modify the way your images look in a gallery or in specific sections of your application you can add a few classes to make them stand out.

//display with rounded corners of 2px
<img class="img-rounded" alt="" src="img/users/alex.jpg"/>
//display as a circle
<img class="img-circle" alt="" src="img/users/alex.jpg"/>
//display as a thumbnail with border
<img class="img-thumbnail" alt="" src="img/users/alex.jpg"/>

Forms

What structure do my forms need to have?

ArkAdmin does not impose any new structure to your information or forms. Instead it relies on the structure provided within Bootstrap 3 Forms.

Here's an example on how to display a simple form.

<form role="form">
    <div class="form-group">
        <label for="exampleInputEmail1">Email address</label>
        <input type="email" class="form-control" id="exampleInputEmail1" placeholder="Enter email">
    </div>
    <div class="form-group">
        <label for="exampleInputPassword1">Password</label>
        <input type="password" class="form-control" id="exampleInputPassword1" placeholder="Password">
    </div>
    <button type="submit" class="btn btn-primary">Submit</button>
</form>

There are also a few classes in Bootstrap that allow you to change how the elements are displayed within the form.

//add the .form-horizontal class to display label next to the inputs
<form class="form-horizontal" role="form">
//add the .form-inline class to display elements inline
<form class="form-inline" role="form">

For checkboxes you can use them as any other input, but when they are rendered they will look consistently on any modern browser.

//normal checkbox in a form
<div class="checkbox">
    <label>
        <input type="checkbox" value="" />
        Option one is this and that—be sure to include why it's great
    </label>
</div>
//inline checkbox
<label class="checkbox-inline">
    <input type="checkbox" id="inlineCheckbox1" value="option1"> 1
</label>

Radio boxes are the same as checkboxes, but you have to use a different class in order to render them.

<div class="radiobox">
    <label>
        <input type="radio" name="optionsRadios" id="optionsRadios1" value="option1" checked />
        Option one is this and that—be sure to include why it's great
    </label>
</div>

Select elements are enhanced with the Select2 jQuery plugin.
This allow you a bit more control over how you load data, or in which element you save the selected option.
Please check out the documentation of the plugin for all the options that are available to you.

The plugin in not applied directly to all your select elements. You can configure that as well in the ark.js file, but the theme applies it by default only to elements with the .select2 class.

<select class="form-control select2">
    <option value=""> </option>
    <option value="AK">Alaska</option>
    <option value="AL">Alabama</option>
    ...
</select>

Another cool thing about ArkAdmin is that it integrates a file upload plugin that makes all your file inputs look consistent and you can also have a preview before uploading any images.

//default file input type
<div class="fileinput fileinput-new" data-provides="fileinput">
    <span class="btn btn-default btn-file">
        <span class="fileinput-new">Select file</span>
        <span class="fileinput-exists">Change</span>
        <input type="file" name="..."/>
    </span>
    <span class="fileinput-filename"></span>
    <i class="fa fa-times fileinput-exists" data-dismiss="fileinput"></i>
</div>

//with image preview
<div class="fileinput fileinput-new" data-provides="fileinput">
    <div class="fileinput-new thumbnail">
        <img alt="" src="img/no-image-selected.png"/>
    </div>
    <div class="fileinput-preview fileinput-exists thumbnail" style="max-width: 200px; max-height: 150px;"></div>
    <div >
        <span class="btn btn-info btn-file">
            <span class="fileinput-new">Select image</span>
            <span class="fileinput-exists">Change</span>
            <input type="file" name="..."/>
        </span>
        <a href="#" class="btn btn-info fileinput-exists" data-dismiss="fileinput">Remove</a>
    </div>
</div>

This may look a bit too much, but it allows you to modify the code in any way that you need to, by only knowing how the .fileinput-* classes work.
Apply the .fileinput-new class to elements that you want displayed when the user did not select a file yet, and the .fileinput-exists class for elements that should display when the file was selected.
The .fileinput-preview class will load a preview of the image loaded by the user, while the .fileinput-filename will load the name of the selected file.

The plugin used to obtain this functionality is documented online at Jasny Bootstrap File Input.

Plugins

Integrated plugins and how to use them

ArkAdmin uses the following plugins or external libraries to obtain all it's functionality.

  1. jQuery - the basis for everything in bootstrap
  2. jQuery UI
  3. jQuery Uniform Plugin - for formatting checkboxes and radio buttons
  4. Jasny Bootstrap - for the file inputs and input masks
  5. jQuery Autosize - for making the textareas expand automatically
  6. MomentJs - for working easily with date and time values
  7. Date Range Picker - ease selection of date ranges
  8. Date Picker - for working with date fields
  9. jQuery EasyPieChart - for the small charts on the homepage
  10. jQuery Flot - for all the other charts in the theme
  11. Select2 - for improving the default select elements
  12. Full Calendar - for the full page calendar plugin
  13. Nestable - for the nested lists page
  14. Prettify - for all the above code previews
  15. CKEditor - for wysiwyg editor

In order to ensure consistent design ArkAdmin adds it's own css on top of these plugins.
This gives you the beautiful clean design that you see before you, without any elements out of place.

The src/js/ark.js file contains the setup for all the plugins listed above as well as any initiation code for the theme.
It is advised that you use copy this file in the public/scripts/js folder and modify it to your own needs.
You can remove any plugins that you don't use for your application and all the examples.

We will go through all the initiation scripts that are available in the ark.js file and list them one by one for you to use in case you want to completely rewrite the provided ark.js file.


ArkAdmin

The bare minimum code required for the ArkAdmin code to run correctly is available in the src/js/ark.base.js file. You can use it to add all the other plugins code to it and make all the plugins work.

The src/js/ark.js contains the full source code for the theme, including the current examples.


UniformJs

UniformJs is applied on all checkboxes and radio buttons directly in the ark.base.js file.

<script type="javascript">
$("input:checkbox, input:radio").uniform();
</script>

EasyPieChart

In order to setup the fancy pie charts that you see on the Dashboard you have to have something like this in your code:

<script type="javascript">
$('.pie-chart').easyPieChart({
    barColor: "#428bca",
    lineWidth: 4,
    animate: 1000,
    onStart: function () {
        if (this.options.isInit) {
            return;
        }
        this.options.isInit = true;
        var color = $(this.el).data('barColor');
        if (color) {
            this.options.barColor = color;
        }
    },
    onStep: function (oldVal, newVal, crtVal) {
        $(this.el).find('span').text(Math.floor(crtVal) + '%');
    }
});
</script>

The associated html code to make this work looks like this:

<div class="pie-chart" data-percent="73"><span>0%</span></div>

Date Range Picker

The date range picker plugin allows you to specify exactly what options the user should have and if they can choose custom values or not.
To set it up use the following code and customize it to your own use case. 

<script type="javascript">
$('#reportrange').daterangepicker(
    {
        ranges: {
            'Today': [moment(), moment()],
            'Yesterday': [moment().subtract('days', 1), moment().subtract('days', 1)],
            'Last 7 Days': [moment().subtract('days', 6), moment()],
            'Last 30 Days': [moment().subtract('days', 29), moment()],
            'This Month': [moment().startOf('month'), moment().endOf('month')],
            'Last Month': [moment().subtract('month', 1).startOf('month'), moment().subtract('month', 1).endOf('month')]
        },
        startDate: moment().startOf('month'),
        endDate: moment().endOf('month')
    },
    function(start, end) {
        $('#reportrange span').html(start.format('MMMM D, YYYY') + ' - ' + end.format('MMMM D, YYYY'));
    }
);
</script>

The associated html code looks like this:

<div id="reportrange" class="pull-right">
    <i class="fa fa-calendar"></i>
    <span>This month</span> <b class="caret"></b>
</div>

Full Calendar

The calendar plugin contains a solid API that lets you interact with the calendar in a lot of ways.
The configuration part depends a lot on the application you are trying to build and the way you want your users to interact with it.
For a complete reference to the API please visit the official documentation.

In order to setup a simple version of the calendar you can just initialize the plugin on your element with the list of events that should be displayed.

// the code that should be added to your page in order to display the events

var d = date.getDate();
var m = date.getMonth();
var y = date.getFullYear();

var calendar = $('#calendar').fullCalendar({
    header: {

        right: 'month,agendaWeek,agendaDay'

    selectable: true,
    editable: true,
    events: [
        {
            title: 'All Day Event',
            start: new Date(y, m, 1)
        },
        {
            title: 'Long Event',

            end: new Date(y, m, d-2)

        {
            id: 999,
            title: 'Repeating Event',
            start: new Date(y, m, d-3, 16, 0),
            allDay: false
        },
        {
            title: 'Meeting',

        }
    ]
});
</script>

In order to initialize the charts for an element you have to write something like this:

<script type="javascript">

Wysiwyg Editor

How do I edit something and the end view to be the same?

If you want to view something very similar to the end result while the document is being created, you can use the following code:

<textarea  class="ckeditor" cols="80" rows="10" full-ckeditor="true">
</textarea>

ArkAdmin also contains inline wysiwyg editor. You can use the following code for it:

<div class="ckeditor" contenteditable="true" basic-ckeditor="true">
</div>

full-ckeditor and basic-ckeditor attributes can be used to set CKEditor toolbar options.

Google Maps

How do I include Google Maps in my application?

If you want to include Google Maps, you can use the following code:

<div class="map-canvas" map-zoom="11" map-latitude="40.6862611" map-longitude="-74.0115122">
</div>

map-zoom, map-latitude and map-longitude attributes can be used to set the zoom depth, the latitude and the longitude of map focus point. Setting terrain-map or styled-map attributes to true you can change the map appearence.
